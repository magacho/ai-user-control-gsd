# Phase 02.4: Padronização de UI — Ícones, Nomenclatura e Visão por Licença - Context

**Gathered:** 2026-02-28
**Status:** Complete (retroactive formalization)

<domain>
## Phase Boundary

Fase intermediária de refinamento pós-02.3 que entrega três capacidades:
1. **Documentação formal** de todas as regras de negócio do sistema (login, inatividade, mapeamento)
2. **Padronização de nomenclatura** — "GitHub Copilot" → "Copilot" em toda a aplicação (DB + código + UI)
3. **Identidade visual por ferramenta** — ícones SVG, displayName no enum, e reframe completo da área de usuários para mostrar licenças ao invés de status/lastLogin

Esta fase não adiciona funcionalidade nova — consolida e melhora a apresentação do que já existe.
</domain>

<decisions>
## Implementation Decisions

### Nomenclatura de Ferramentas
- "GitHub Copilot" renomeado para "Copilot" em toda UI (mais limpo, alinhado com branding atual)
- Migration V11 atualiza registros existentes no banco
- Enum `GITHUB_COPILOT` mantém o nome técnico, mas `displayName` retorna "Copilot"

### Identidade Visual (SVG Icons)
- Cada `AIToolType` agora possui `displayName` e `iconPath`
- Ícones SVG minimalistas em `/static/images/` (claude.svg, copilot.svg, cursor.svg, custom.svg)
- Ícones usados em 5x5px na UI — escalam sem perda de qualidade

### Reframe da Área de Usuários
- Coluna "Status" removida da listagem → substituída por "Licenças" (ícones das ferramentas)
- Coluna "Last Login" removida da listagem (informação operacional, não relevante para visão de inventário)
- Filtro por status substituído por filtro "Ferramenta" (dropdown com todos os AIToolType)
- Batch loading de tool accounts via `findByUserIdIn()` para evitar N+1
- `UserSpecification.hasToolType()` usa JPA join em toolAccounts → aiTool
- `User` entity ganhou `@OneToMany toolAccounts` para habilitar o join

### Documentação de Regras de Negócio
- Estrutura em `docs/` com índice central (`BUSINESS_RULES.md`) e regras por domínio
- Três domínios: LOGIN (BR-LOGIN-01..06), INACTIVITY (BR-INACT-01..11), MAPPING (BR-MAP-01..16)
- Regras derivadas do código existente, não de novos requisitos

### Claude's Discretion
- Cores dos ícones SVG escolhidas para consistência com branding das ferramentas
- Ordem dos ícones na coluna Licenças segue ordem dos accounts do usuário
</decisions>

<specifics>
## Specific Ideas

- O reframe da área de usuários muda a filosofia: de "gestão operacional" (quem está ativo/quando logou) para "inventário de licenças" (quem tem acesso a quê)
- Status e lastLogin continuam disponíveis na página de detalhe do usuário
- A documentação de regras serve como referência para desenvolvimento futuro e onboarding
</specifics>

<deferred>
## Deferred Ideas

- Tooltip nos ícones com detalhes da licença (status da conta, data do último uso)
- Exportação do inventário de licenças (CSV/Excel)
- Contagem de ferramentas por usuário como métrica no dashboard
- Badge de compliance na listagem (ex: "3 ferramentas, 1 inativa há 60+ dias")
</deferred>
