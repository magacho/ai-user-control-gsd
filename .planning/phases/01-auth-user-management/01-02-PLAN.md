  <action>
    **1. Implement SecurityConfig.java (replace existing JWT-based stub):**

    `@Configuration @EnableWebSecurity @RequiredArgsConstructor` class with:
    - Inject `CustomOidcUserService`
    - `@Bean SecurityFilterChain filterChain(HttpSecurity http)`:
      - `authorizeHttpRequests`: permit `/login/**`, `/oauth2/**`, `/error/**`, `/webjars/**`, `/css/**`, `/js/**`, `/images/**`, `/actuator/health`; all other requests require `hasRole("ADMIN")`
      - `oauth2Login`: custom login page at `/login`, `userInfoEndpoint` with `oidcUserService(customOidcUserService)`, `defaultSuccessUrl("/dashboard", true)`, `failureHandler(oAuth2AuthenticationFailureHandler)`
      - `logout`: `logoutSuccessUrl("/login?logout=true")`, `deleteCookies("JSESSIONID")`, `invalidateHttpSession(true)`
      - `sessionManagement`: `maximumSessions(1)`, `maxSessionsPreventsLogin(false)` (new login displaces old)
    - `@Bean HttpSessionEventPublisher` for session event management

    **AUTH-03 simplified per CONTEXT.md:** Only ADMIN role. `hasRole("ADMIN")` on all protected endpoints. No DEVELOPER role distinction needed. Phase 8 (Developer Self-Service) is dropped.

    **2. Implement CustomOidcUserService.java:**

    `@Component @RequiredArgsConstructor` extending `OidcUserService`. Override `loadUser(OidcUserRequest)`:
    1. Call `super.loadUser(userRequest)` to get `OidcUser`
    2. Extract email from `oidcUser.getEmail()`
    3. **Domain validation (USER-05):** Reject if email is null or does not end with `@bemobi.com`. Also validate `oidcUser.getClaimAsString("hd")` equals `bemobi.com`. Throw `OAuth2AuthenticationException` with error code `"invalid_domain"` and user-friendly message.
    4. **Pre-registration check:** Look up user in DB via `userRepository.findByEmail(email)`. Check if email is in `appProperties.getAdminEmails()`.
    5. If user not in DB AND not a config admin: throw `OAuth2AuthenticationException` with code `"user_not_registered"` and message "Your account has not been registered. Contact an administrator."
    6. If user not in DB BUT is config admin: auto-create User entity with email, name from `oidcUser.getFullName()`, avatarUrl from `oidcUser.getPicture()`, status ACTIVE. Save via repository.
    7. **Deactivated check:** If `user.getStatus()` is INACTIVE or OFFBOARDED, throw `OAuth2AuthenticationException` with code `"user_deactivated"` and message "Your account has been deactivated. Contact an administrator."
    8. **Profile sync on every login:** Update user.name, user.avatarUrl, user.lastLoginAt from Google. Save.
    9. **Assign ADMIN authority:** Create new authority set with `ROLE_ADMIN`. Return `DefaultOidcUser(authorities, idToken, userInfo)`.

    **3. Implement OAuth2AuthenticationSuccessHandler.java:**

    `@Component` implementing `AuthenticationSuccessHandler`. On success, redirect to `/dashboard`. Log successful login with user email.

    **4. Implement OAuth2AuthenticationFailureHandler.java:**

    `@Component` implementing `AuthenticationFailureHandler`. Extract error code from the `AuthenticationException` (which wraps `OAuth2AuthenticationException`). Map error codes to URL parameters:
    - `invalid_domain` -> `/login?error=domain`
    - `user_not_registered` -> `/login?error=unregistered`
    - `user_deactivated` -> `/login?error=deactivated`
    - default -> `/login?error=true`
    Redirect to the mapped URL.

    **5. Create login.html:**

    Minimal centered card layout (Tailwind CSS via CDN):
    - Include `<script src="https://cdn.tailwindcss.com"></script>`
    - App name "AI User Control" as heading
    - Tagline "AI Tool Usage Management"
    - Single "Sign in with Google" button linking to `/oauth2/authorization/google`
    - Conditional error messages based on `?error=` parameter:
      - `domain`: "Access is restricted to @bemobi.com accounts."
      - `unregistered`: "Your account has not been registered. Contact an administrator."
      - `deactivated`: "Your account has been deactivated. Contact an administrator."
      - `true` (generic): "Authentication failed. Please try again."
    - Conditional success message for `?logout=true`: "You have been logged out."
    - Clean, minimal Linear/Vercel aesthetic. Desktop-only. No responsive design needed.

    **6. Create error/access-denied.html and error/error.html:**

    access-denied.html: Friendly page stating "You do not have permission to access this page" with a link back to dashboard or login.

    error.html: Generic error page with Thymeleaf error status and message display. Clean Tailwind styling.
