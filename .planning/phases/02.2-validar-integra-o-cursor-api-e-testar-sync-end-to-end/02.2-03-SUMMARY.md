## Decisions Made
- JOIN FETCH is the correct fix (not @Transactional on the controller) because spring.jpa.open-in-view=false means the session closes after the repository call; the data must be eagerly loaded in the query itself
- Used LEFT JOIN FETCH for user relationship (nullable -- external accounts have no user) and JOIN FETCH for aiTool (non-null, always exists)
- COUNT queries for sidebar badge because SidebarModelAdvice only needs the count, not the full entity graphs with all relationships
