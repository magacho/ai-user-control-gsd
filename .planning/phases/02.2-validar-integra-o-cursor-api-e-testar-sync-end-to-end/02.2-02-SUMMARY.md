## Decisions Made
- ToolSyncDetail is populated in two phases: seatsFound and error during parallel fetch, then linked/unmatched/suspended/revoked during the link phase. This avoids needing to pass link results back through the fetch pipeline.
- Per-tool breakdown is shown in both success and error toasts, so admins see which tools succeeded even during partial failures.
- Added getToolDetail() to the Builder class to enable cross-phase enrichment of the same ToolSyncDetail object without needing a separate map.
