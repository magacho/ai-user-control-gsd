## Decisions Made
- Used OffsetDateTime.parse() for GitHub API dates (they include timezone offsets like -06:00, not pure UTC)
- Per-field try-catch wrapping for date parsing to avoid crashing entire seat on a single malformed date
- createdAtSource uses write-once semantics (only set if null) while lastActivityAt always overwrites with non-null values
- GWS git_name query uses sanitizeQueryValue to escape single quotes for query safety
